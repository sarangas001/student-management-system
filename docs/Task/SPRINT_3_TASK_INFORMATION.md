# Testing Process

For every assigned module:

## 1. Functional Testing

Verify all available features work as expected.

Examples:

* Create records
* Update records
* Delete records
* Search records
* Filter records
* Submit forms

---

## 2. UI Testing

Verify:

* Layout consistency
* Design accuracy
* Alignment issues
* Responsive behavior
* Mobile compatibility

---

## 3. Validation Testing

Verify:

* Required fields
* Invalid inputs
* Error messages
* Form restrictions

---

## 4. API Integration Verification

Verify:

* Data loads successfully
* Data updates correctly
* Error responses handled properly
* Loading states appear correctly

---

## 5. Bug Reporting

If a bug is found:

Create a GitHub Issue including:

### Title

```text
[BUG] Student Attendance Page Not Loading Data
```

### Description

```text
Module:
Student Attendance

Steps to Reproduce:
1. Login as Student
2. Open Attendance Page
3. Select Course

Expected Result:
Attendance records should load

Actual Result:
Loading spinner remains forever

Screenshots:
Attach screenshot

Environment:
Chrome Version XX
```
